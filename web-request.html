⭐ Classic interview question

"How does the Ingress Controller know a Pod crashed so it stops sending traffic to it?"

Answer: Through an API Server Watch Loop.

When a Pod fails its Readiness Probe, the Endpoints Controller (on the Masternode) detects this and removes the Pod's IP from the Endpoints object in etcd. The Ingress Controller is constantly "watching" the API Server for these changes. The moment the API server updates, the Ingress Controller instantly updates its internal NGINX configuration in memory, dropping the bad IP.
