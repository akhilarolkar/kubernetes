⭐ Classic interview question

"Does the Kubernetes API Server store pod logs in etcd?"

Answer: No! Logs are strictly stored on the Worker Node's physical disk.

The API Server simply acts as a proxy. It authenticates you, finds out which node has your pod, and forwards your request to that node's kubelet. If the pod dies or the worker node crashes, those logs are permanently deleted unless you have an external log aggregation tool (like Fluentd + ElasticSearch, or Promtail + Loki) scraping them.
